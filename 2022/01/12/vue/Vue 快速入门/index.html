    VUE 简介

    什么是VUE

    1. 构建用户界面
      • 用vue 往html 页面中填充数据，非常的方便
    2. 框架
      • 框架是一套现成的解决方案
      • 要学习vue，就是学习vue 框架中的规定的用法！

    VUE 的特性

    vue框架的特性，主要体现在如下两方面：

    1. 数据驱动视图
    2. 双向数据绑定

      在网页中，form表单负责采集数据，ajax 负责提交数据

      • js 数据的变化，会被自动渲染到页面上
      • 页面上表单采集的数据发生变化的时候，会被vue 自动获取到，并更新到 js 数据中

    1. uve 基本使用步骤

    1. 导入 vue.js 的 script 脚本文件
    2. 在页面中 声明一个将要被vue所控制的DOM区域
    3. 创建vm实例对象（vue 实例对象)

    uve 的指令与过滤器

    1. 指令的概念

    指令（Directives）是vue 为开发中提供的模板语法，用于 辅助开发者渲染页面的基本结构

    vue 中的指令按照不同的用途可以分为如下6大类：

    1. 内容渲染
    2. 属性绑定
    3. 事件绑定
    4. 双向绑定
    5. 条件渲染
    6. 列表渲染
    1.1 内容渲染指令

    内容指令用来辅助开发者渲染DOM元素的文本内容。常用的内容渲染指令有如下3个：

    • v-text 指令的缺点：会覆盖元素内部原有的内容！
    • { { } } 差值表达式：在实际开发中用的最多，只是内容的占位符，不会覆盖原有的内容！
    • v-html 指令的作用：可以把带有标签的字符串，渲染成真正的HTML内容！
    v-text

    缺点：会覆盖元素内默认的值

    { {} } （插值表达式-mustache）

    开发中最常使用

    v-html

    如果要把包含HTML标签的字符串渲染为页面的HTML元素，则需要用到v-html 这个指令

    1.4双向绑定指令 v-model

    1. input 输入框
    2. textarea
    3. select

    v-model 绑定到 value 属性
    监听绑定到的元素值，当发生改变时更新model

    v-model指令的修饰符
    修饰符 作用 示例
    .number 自动将用户的输入值转为数值类型 <input v-model.number="age"/>
    .trim 自动过滤用户输入的首尾空白字符 <input v-model.trim="msg"/>
    .lazy 在“change”时而非“input”时更新 <input v-model.lazy="msg"/>

    1.5 条件渲染指令

    • v-if 每次创建或移除元素，来显示或隐藏元素
    • v-show 动态添加或移除 display:none 样式，来显示或隐藏元素

    频繁切换元素显示状态用v-show性能更好

    如果刚进入界面某些元素就不需要被展示，v-if性能更好

    1.6 列表渲染指令 v-for

    vue 提供了v-for 列表渲染指令，用来辅助开发者基于一个数组来循环渲染一个列表结构。v-for 指令需要使用 itme in items 形式的特殊语法，其中：

    • items 是待循环的数组
    • item 是被循环的每一项

    key 的注意事项

    1. key 的值只能是字符串或数字类型
    2. key 的值必须具有唯一性
    3. 建议把数据项id属性的值作为key的值
    4. 使用index的值当作key的值没有任何意义
    5. 建议使用v-for指令是一定要指定key的值（既提升性能、又防止列表状态紊乱）

    过滤器

    过滤器的注意点

    1. 要定义到 filters节点下，本质是一个函数
    2. 在过滤器函数中 ，一定要有return值
    3. 在过滤器的形参中，就可以获取到过滤器前的参数
    4. 如果全局过滤器和私有过滤器名字一致，此时按照“就近原则”，调用的是“私有过滤器”

    vue-li

    1.单页面应用程序

    什么是单页面应用程序（single page application）简称spa，指 一个web网站中只有唯一的一个HTML页面 ，所以交互都在这唯一的页面完成。

    2. 什么是vue-cli

    vue-cli 是vue.js开发的标准工具。它简化了程序员基于webpack创建工程化的Vue项目的过程。

    引用自 vue-cli 官网上的一句话
    程序员可以专注在撰写应用上，而不必花好几天去纠结 webpack 配置问题

    中文官网： https://cli.vuejs.org/zh/

    3. 安装和使用

    vue-cli 是 npm 上的一个全局包，使用 npm install 命令，即可方便的把它安装到自己的电脑上

    基于vue-cli 快速生成工程化的项目

    方向键上下选择项目

    vue 项目中src目录的构成：

    1. assets 文件夹：存放项目中静态资源
    2. components 文件夹：程序员封装的、可复用的组件，都要放到components目录下
    3. main.js 是项目的入口文件。整个项目的运行最先执行的文件 、
    4. app.vue 是项目的跟组件
    4.vue项目的运行流程

    在工程化的项目中，vue要做的事情很单纯：通过main.js把app.vue渲染到index.html的指定区域中。

    其中：

    1. App.vue用来编写待渲染的模板结构
    2. index.html中需要预留一个el区域
    3. main.js把App.vue渲染到了 index.html所预留的区域中

    6. 组件之间的样式冲突问题

    默认情况下，写在.vue组件中的样式会全局生效，因此很容易造成多个组件之间的样式冲突问题。

    导致组件之间样式冲突的根本原因是：

    1. 单页面应用程序中，所有组件的DOM结构，都是基于唯一的index.html页面进行呈现的
    2. 每个组件中的样式，都会**影响整个index.html页面中的DOM元素
    6.1 sytle添加scoped解决这个问题
    6.2 style /deep/

    当使用第三方组件库的时候，如果有修改第三方组件默认样式的需求，需要用到 /deep/

    5. 组件的生命周期

    5.1 生命周期＆生命周期函数

    生命周期（life cycle） 是指一个组件从创建-》运行-》销毁的整个阶段，强调的是一个时间段

    生命周期函数：是有vue框架提供的内置函数，会伴随这组件的生命周期，自动按次序执行。

    5.3 组件生命周期函数的分类

    创建阶段：beforeCreate->created->beforeMount->mounted
    组件运行阶段：beforeUpdate->updated
    组件销毁阶段：beforeDestroy->destroy
